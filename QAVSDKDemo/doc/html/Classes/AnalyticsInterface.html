+ onMergeUserAction:isSucceed:elapse:size:params:

事件合并统计上报接口 主要是对同一事件在内存中进行合并统计 合并统计最终数据会在上报事件的基础数据和附加数据中体现

+ (BOOL)onMergeUserAction:(NSString *)eventName isSucceed:(BOOL)isSucceed elapse:(long)elapse size:(long)size params:(NSDictionary *)params

Discussion

事件合并统计上报接口 主要是对同一事件在内存中进行合并统计 合并统计最终数据会在上报事件的基础数据和附加数据中体现

最终上报事件基础数据为： EventName:统计事件名 EventTime：统计起始时间 SrcIp:统计第一次终端ip，累计过程不更新 CosumeTime：总耗时，elapse的累加，不区分成功与否 EventResult:统计的第一次成功与否的值，累计过程不更新 PackageSize：总大小，size的累加，不区分成功与否 最终上报事件附加数据为： rqdUT：事件统计更新时间（EventUpdateTime），也就是最后一个事件的时间 rqdEC：事件总次数(EventCount) rqdTE：成功的总耗时(TrueEapse) rqdTS：成功的上报总size(TrueSize) rqdFC：失败的次数(FalseCount) rqdFE：失败的总耗时(FalseElapse) rqdFS：失败的上报总size(Falsesize)

事件上报中统计事件附加参数params，主要是一个key－value的对象 比如相应统计一个事件的请求IP的次数，可以每次使用 “$ip->1"这样最终合并的事件中会由 ”$ip -> n"这样的数据 比如统计一个key的数据之和，可以使用 “$key -> num1",最终会有”$key -> （num1 ＋ 。。。。）"这样的数据 *
